Você é um Diretor de Arte sênior (luxury editorial / minimal / Apple-like). Sua tarefa é **recriar o visual do SaaS inteiro** (todas as telas existentes) com um **novo conceito premium** — sem mudar nenhuma funcionalidade.

REGRAS INEGOCIÁVEIS

* NÃO alterar rotas, fluxos, estrutura, hierarquia das telas, conteúdos, nomes de campos ou CTAs.
* NÃO remover telas nem inventar etapas novas.
* Apenas redesign visual: tipografia, cores, espaçamento, grid, componentes, estados, ícones, bordas, sombras.
* Visual obrigatório: **fundo branco/off-white**, estética **moda/editorial**, minimalista, sofisticada.
* Proibido: neon, cores saturadas, gradientes chamativos, brilho/gloss, sombras pesadas, “cara de template genérico”.

DIREÇÃO CRIATIVA (conceito)

* “Luxo silencioso” (quiet luxury): elegância discreta, muito respiro, tipografia impecável.
* Alto contraste com suavidade: preto “ink” + cinzas bem calibrados + **apenas 1 cor de acento sofisticada e discreta**.
* O premium vem do layout: grid, alinhamento, ritmo e microdetalhes — não de enfeites.

ENTREGÁVEIS NO FIGMA (faça tudo dentro do arquivo)

1. Crie uma página: **Design System — Foundations**

   * Paleta: off-white/white + neutros + 1 acento sofisticado (muted) + estados (success/warning/error/info) discretos.
   * Tipografia: escolha uma tipografia UI moderna (sans) e, se fizer sentido, uma tipografia editorial para títulos (serif bem elegante) — ou use 1 família MUITO bem escalada.
   * Defina escala completa: H1–H6, body, small, caption com line-height e tracking.
   * Spacing system (8pt), grid (desktop), radius (poucos níveis), bordas hairline, sombras quase invisíveis (2 níveis no máximo), ícones (mesmo stroke e tamanhos 16/20/24).
   * Transforme tudo em estilos/variáveis/tokens do Figma.

2. Crie uma página: **Design System — Components**
   Construa componentes com Auto Layout + variantes + estados:

   * Buttons: primary/secondary/ghost/link/danger + hover/pressed/disabled/loading
   * Inputs: text, textarea, select, search + estados (default/focus/error/disabled)
   * Checkbox/radio/switch
   * Tabs, dropdown/menu
   * Tables: header/row/hover/selected/empty + paginação
   * Modal/Drawer, Toast/Alert
   * Badge/Tag, Card, Empty states
   * Navegação (sidebar/topbar) no estilo premium

3. NOVO CONCEITO (sem referência externa)
   Antes de aplicar em tudo, crie 2 mini-direções internas no arquivo:

   * Direção A: “Quiet Luxury Editorial” (mais moda, mais respiro, títulos com presença)
   * Direção B: “Clean Tech Premium” (mais Apple, mais precisa e clean)
     Para cada direção, mostre: mini paleta + tipografia + exemplo de botão + card + tabela.
     Depois escolha a melhor (a mais premium e autêntica) e siga com ela.

4. APLICAÇÃO GLOBAL (sem quebrar nada)
   Aplique o novo design system em **todas as telas existentes**:

   * Atualize botões, campos, tabelas, cards, navegação, espaçamentos, headings e ícones.
   * Garanta consistência absoluta entre telas.
   * Garanta estados (hover/focus/disabled/loading/erro) onde fizer sentido.
   * Resultado final deve parecer **produto de luxo**: minimal, elegante, editorial — sem aparência genérica.

5. FINALIZAÇÃO

   * Gere uma lista: “Telas atualizadas” + “10 regras do sistema visual” (pra manter consistência).
   * Aponte 10 melhorias rápidas de UI (somente visual) que elevem o premium (respiro, hierarquia, alinhamento, microcopy discreta).
