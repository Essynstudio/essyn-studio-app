0) Princípios que não podem quebrar (base da galeria premium)

Rápida no celular (carregar em 2–3s, sempre)

Sem fricção pro cliente (entrar, ver, favoritar, baixar)

Controle total pro fotógrafo (acesso, expiração, versões, logs)

Entrega profissional (organização, seleção, aprovação, download certo)

Segura e rastreável (token, logs, watermark, anti-abuso)

Bonita e “brandável” (aparência premium = percepção de valor)

1) Objetos (o que existe no sistema por trás)

Você precisa modelar isso, senão vira gambiarra:

Studio/Conta (marca do fotógrafo, domínio, regras padrão)

Cliente (nome, e-mail/WhatsApp, tags, eventos)

Projeto/Evento (casamento, ensaio, formatura — conecta tudo)

Galeria (a entrega em si; tem capa, regras, link)

Coleções/Pastas (Cerimônia, Festa, Making Of…)

Asset (Foto/Vídeo) + Variants (Web / Alta / Thumb / Preview)

Favoritos (por usuário, por lista, por pessoa)

Comentários (com marcação + status)

Aprovação (quando cliente finaliza seleção)

Política de Download (liberado, limitado, pago, expirado)

Álbum/Impressão (opcional): Itens, pedidos, entrega

Logs/Atividades (quem viu, baixou, compartilhou, quando)

Tokens de acesso (link público, link privado, convites)

2) Papéis e permissões (essencial)

Admin (fotógrafo): tudo

Equipe (assistente/segundo fotógrafo): upload/organizar (sem ver financeiro, se quiser)

Cliente principal: ver, favoritar, comentar, aprovar, baixar (conforme regra)

Convidado: ver/favoritar/baixar (limitado)

Lab/Fornecedor (opcional): receber seleção do álbum / arquivos específicos

Diferencial: favoritos e comentários separados por pessoa (noiva / noivo / mãe / etc).
