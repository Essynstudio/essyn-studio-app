# Checklist: Self-Critique

> Todo builder DEVE executar ANTES de reportar "done" ao Lead.

## Completude
- [ ] Reli os critérios de "done" do spawn prompt — TODOS atendidos?
- [ ] A spec foi implementada completa, não parcialmente?
- [ ] Edge cases estão tratados (null, undefined, empty)?

## Qualidade do Código
- [ ] Reli cada função — entenderia em 30 segundos daqui 3 meses?
- [ ] Nomes descritivos (sem abreviações crípticas)?
- [ ] Nenhum código duplicado que deveria ser extraído?
- [ ] Error handling explícito (sem catch vazio)?

## Testes
- [ ] Happy path testado?
- [ ] Pelo menos 1 edge case?
- [ ] Pelo menos 1 error case?
- [ ] Todos passam localmente?

## Limpeza
- [ ] Nenhum console.log / debug statement?
- [ ] Nenhum código comentado restante?
- [ ] Nenhum TODO sem justificativa?
- [ ] Imports não utilizados removidos?

## Autocrítica Obrigatória

Identifique pelo menos 2 pontos de melhoria:
1. **Ponto 1:** [descreva + corrija ANTES de reportar]
2. **Ponto 2:** [descreva + corrija ANTES de reportar]
